## 3. Jobscan Scan Results — Categories They Check

### Keyword Analysis (5 categories)
1. **Hard Skills** — technical skills from job description
2. **Soft Skills** — interpersonal/leadership skills
3. **Other Keywords** — industry terminology, tools
4. **Buzzwords** — overused cliches to avoid
5. **Industry Lingo** — domain-specific vocabulary

### ATS Searchability Checks
- ATS-specific tips (detects which ATS the company uses)
- Contact Information (physical address, email, phone)
- Summary section presence
- Section Headings (education, work experience, work history)
- Job title match

### Recruiter Tips
1. Job level match
2. Measurable results (quantified achievements)
3. Resume tone
4. Web presence
5. Word count

### Formatting Analysis
1. Layout (tables, images, paragraph length, text alignment)
2. Font & font size (special characters, bold styling, font consistency, ATS readability)
3. Page setup (headers/footers, margins, page size)

## 6. Prioritized Recommendations

### TIER 1 — Do Now (High Impact, Medium Effort) 🔥

#### 1.1 Granular Scan Result Categories
**What:** Break our single keyword match into 5 separate scores like Jobscan: Hard Skills, Soft Skills, Formatting, Searchability, Recruiter Tips.
**Why:** This is the #1 thing that makes Jobscan's results look more professional and actionable than ours. Users see 5 progress bars with specific issue counts instead of one number.
**How:** We already extract keywords by category in `keyword_engine.py`. Surface these as separate scored sections in the results UI.
**Impact:** HIGH — makes scan results dramatically more useful
**Effort:** MEDIUM — mostly frontend + restructuring existing data
**Constraint:** No account needed — all client-side display

#### 1.2 Cover Letter Generator (Free, One-Time)
**What:** Add "Generate Cover Letter" button after scan. AI writes a tailored cover letter using resume + job description.
**Why:** This is Jobscan's paid-only feature. We can offer a basic version free (or bundle with CV builder payment). Huge value-add that keeps users on our site longer.
**How:** New endpoint `/api/build/cover-letter`. Single Claude API call. Display in-browser with copy/download.
**Constraint:** No account — generate and display instantly. Could be rate-limited to 3/day.

#### 1.3 AI Bullet Point Enhancer
**What:** "Improve this bullet point" feature. User pastes a weak bullet → AI rewrites it with metrics and action verbs.
**Why:** Jobscan charges $30+/mo for this. We can offer a limited free version.
**How:** Simple Claude API call. Can live on the CV builder page or as a standalone micro-tool.
**Constraint:** Rate limit to prevent abuse. No account needed.

### TIER 2 — Do Next (High Impact, Moderate Effort) 🎯

#### 2.1 Resume Summary Generator
**What:** Upload resume + target job → AI generates a professional summary paragraph tailored to the role.
**Why:** Everyone struggles with the summary section. Jobscan gates this behind premium.
**How:** Claude API call with resume context + job description. Display editable result.
**Constraint:** Bundle with scan results as "bonus" output.

#### 2.2 Measurable Results / Quantification Check
**What:** Flag bullet points that lack numbers/metrics. Suggest where to add quantification.
**Why:** Jobscan's "Recruiter Tips" section checks this. It's one of the most actionable pieces of feedback.
**How:** Regex + AI analysis to detect bullets without numbers/percentages/metrics.
**Impact:** HIGH — directly actionable, differentiating
**Effort:** LOW — pattern matching + AI enhancement of existing suggestions

#### 2.3 Job Level Match Detection
**What:** Detect if the resume's experience level matches the job description's seniority.
**Why:** Applying for wrong-level jobs is a huge hidden reason for rejections.
**How:** AI classifies both resume and job into entry/mid/senior/lead/executive, flags mismatches.
**Impact:** MEDIUM — saves users from wasting applications
**Effort:** LOW — single AI classification call

#### 2.4 SEO Content Pages (Programmatic)
**What:** Build `/resume-examples/{job-title}` and `/skills/{job-title}` pages.
**Why:** This is Jobscan's primary organic traffic driver — 200+ pages targeting high-intent searches. We have zero SEO content right now.
**How:** Generate 20-30 initial pages for top job titles. Each page has: example resume sections, key skills list, ATS tips, CTA to scan.
**Constraint:** Static pages, no user data. Can be generated once and served as content.
**Impact:** HIGH — organic traffic acquisition
**Effort:** MEDIUM — content generation + page templates

### TIER 3 — Future Roadmap (Medium Impact, Higher Effort) 🗺️

#### 3.1 Career Path Discovery
**What:** Upload resume → AI suggests alternative career paths based on transferable skills.
**Why:** Unique, high-engagement feature. Jobscan has this but it's basic.
**How:** Claude API analyzes skills → maps to O*NET or similar career database → suggests 3-5 paths with skill gap analysis.
**Impact:** MEDIUM — great for engagement and virality
**Effort:** MEDIUM

#### 3.2 Post-Interview Thank You Note Generator
**What:** Quick AI tool: enter company name, interviewer name, role, key discussion points → generates professional follow-up email.
**Why:** Low effort, high value micro-tool. Great for SEO ("thank you email after interview" is a massive search query).
**How:** Simple form + Claude API call. Display copyable result.
**Impact:** LOW-MEDIUM — drives SEO traffic, builds brand
**Effort:** LOW

#### 3.3 LinkedIn Profile Tips (Text-Based, No Scraping)
**What:** User pastes their LinkedIn "About" section + target role → AI gives optimization suggestions and rewrites.
**Why:** Jobscan's LinkedIn tool requires profile URL access. We can do a privacy-first version with just pasted text.
**Constraint:** No LinkedIn API, no scraping. User pastes their own text.
**Impact:** MEDIUM
**Effort:** MEDIUM

#### 3.4 Blog / Content Marketing Engine
**What:** Start publishing SEO-optimized blog posts: "How to write a [job title] resume", "Top skills for [industry]", "ATS tips for [specific ATS]".
**Why:** Jobscan has hundreds of blog posts driving organic traffic. We have zero.
**How:** Publish 2-4 posts/month on key resume/ATS topics. Link back to scanner.
**Impact:** HIGH (long-term) — compounds over time
**Effort:** ONGOING

### TIER 4 — Probably Skip (Low Impact or Conflicts with Our Model) ⏭️

| Feature | Why Skip |
|---------|---------|
| Job Application Tracker | Requires persistent user accounts — conflicts with no-login model |
| Chrome Extension | High effort, requires maintaining a browser extension |
| Job Board | Requires job data partnerships, not our core value prop |
| ATS Detection by Company | Requires maintaining a database of which companies use which ATS |
| Resume Manager (multiple versions) | Requires user accounts and persistent storage |
